# Resume Bullet Variants

For each of your 3 projects, this folder has short / medium / long bullet sets keyed to the role archetype the cover-letter templates target. Mix and match: typically you'll pick the bullet set that matches the role, then copy 3–6 bullets per project depending on resume length.

## Files

- `ai_safety_redteam.md` — AI Safety Red-Team project bullets
- `llm_bias_detection.md` — LLM Bias Detection project bullets
- `breast_cancer_classification.md` — Breast Cancer Classification project bullets

## How to use

1. Open the file for the project you want to feature.
2. Pick the **role archetype section** that matches the JD:
   - `## Research / Frontier-lab framing`
   - `## AI Safety / Alignment framing`
   - `## Applied / Production ML framing`
   - `## ML Platform / Infra framing`
   - `## Data Scientist / Quant / Stats framing`
   - `## Civic / Government / Cross-source data framing` *(added because of the NYC OCCECE role)*
3. Copy the **bullet density** that fits your resume real estate:
   - **Short** (2 bullets): tight 1-page resume, project is secondary
   - **Medium** (3–4 bullets): standard 1-page resume, project is primary
   - **Long** (5–6 bullets): 2-page resume or project is the headliner

## Rule of thumb

- Lead each bullet with a verb in past tense.
- Put one number in every bullet. If a bullet has no number, cut it or fold it into one that does.
- The first three words of a bullet should signal what the bullet is about. Recruiters skim, not read.
